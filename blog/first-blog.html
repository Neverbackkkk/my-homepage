神经算子如何求解 PDE：从 DeepONet 到 Koopman Neural Operator

这篇文章整理我对神经算子求解偏微分方程的一次阶段性学习：从“学习一个函数到另一个函数的映射”出发，理解 DeepONet、Fourier 思路和 Koopman Neural Operator 的联系。

1. 为什么关注 PDE 与神经算子

偏微分方程广泛出现在流体、热传导、电磁场和材料模拟中。传统数值方法通常需要网格剖分、离散格式和迭代求解，优点是理论清晰、误差可分析，但在高维问题、复杂几何或大量参数重复求解时，计算成本会很高。

神经算子试图学习的是“算子”而不只是某个有限维向量之间的映射。也就是说，它希望从输入函数，例如初值、边界条件或系数场，直接预测输出函数，例如 PDE 的解。这个角度很适合和算法基础中的抽象、复杂度、泛化能力一起思考。

2. DeepONet：用分支网络和主干网络表示算子

DeepONet 的核心想法是把算子学习拆成两个部分：Branch Net 读取输入函数在若干采样点上的值，提取这个输入函数的特征；Trunk Net 读取待预测位置的坐标，描述输出函数在不同位置上的基函数信息。两者组合后，就可以得到某个位置处的预测值。

从算法角度看，DeepONet 很像是在学习一组可组合的表示：一边压缩输入函数的信息，一边描述输出空间的位置结构。它的优点是概念直观，适合理解“函数到函数”的学习任务；困难在于采样点选择、训练数据覆盖范围和泛化能力都会影响最终效果。

3. Fourier Neural Operator：在频域中捕捉全局结构

Fourier Neural Operator 更强调利用频域表示来建模函数之间的全局关系。它通过傅里叶变换把空间信息转换到频域，在频域中学习主要模态的变换，再回到物理空间。这种方法对于许多具有平滑结构或明显全局模式的 PDE 问题很有吸引力。

这让我联想到算法课程中对表示方式的重视：同一个问题在不同表示空间中难度可能不同。把函数放到频域后，某些全局关联会变得更容易学习，但也需要关注高频信息、边界处理和非周期结构带来的挑战。

4. Koopman Neural Operator：用线性演化理解非线性系统

Koopman 思想的吸引力在于：非线性动力系统在原始空间中很复杂，但如果把状态提升到合适的观测函数空间，系统演化可能近似为线性算子作用。Koopman Neural Operator 就是在神经网络的帮助下学习这种提升空间和演化规律。

对 PDE 求解来说，这种思路提供了一种很有启发性的视角：不一定直接在原始变量上硬拟合复杂动力学，而是寻找一个更适合描述演化的空间。它和算法学习中的“换一个数据结构或状态表示，问题会变简单”有相通之处。

5. 与算法基础学习的联系

学习神经算子并不意味着可以跳过算法基础。相反，排序、递归、动态规划、矩阵运算、复杂度分析和数值稳定性这些基础，会影响我理解模型结构、训练成本和求解误差的方式。一个模型是否有效，不只看神经网络本身，也要看数据组织、离散方式、计算效率和验证指标。

这次用 HTML、CSS 和少量 JavaScript 搭建个人主页，也让我重新体会到“先拆解问题，再逐步实现”的重要性。AI Agent 可以帮助我整理资料、生成网页结构和检查链接，但最终内容是否准确、表达是否真实，仍然需要自己判断。

小结：神经算子为 PDE 快速求解提供了新的思路，但它仍然需要扎实的算法、数值分析和问题建模能力作为基础。
